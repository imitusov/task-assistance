# Task: Implement `db.py`

## Product context
This is a personal Telegram bot that lets multiple independent users manage and
query their own Todoist tasks through natural conversation, powered by an open
LLM with tool calling, hosted on Railway with observability via Grafana. This
module is the **single data-access layer**: every read/write of users,
conversation history, and logs goes through it. All SQL in the entire project
lives here and nowhere else, which is what keeps each user's data isolated and
the schema usage consistent.

- `config.py`: `DATABASE_URL: str`, `MAX_HISTORY_MESSAGES: int` (default `20`),

- `logger.py`:

  - `log_stdout(level: str, event: str, user_id: int | None, data: dict) -> None`

## Database tables used
All three tables, created by module 6 (do not redefine — use as-is):

**users**: `telegram_user_id` BIGINT PK, `todoist_token` TEXT, `language` TEXT
NOT NULL DEFAULT `'en'`, `created_at` TIMESTAMPTZ, `last_active_at` TIMESTAMPTZ.

**conversations**: `id` SERIAL PK, `user_id` BIGINT FK→users ON DELETE CASCADE,
`role` TEXT (user/assistant/tool), `content` JSONB, `created_at` TIMESTAMPTZ.

**logs**: `id` SERIAL PK, `user_id` BIGINT NULLABLE, `level` TEXT, `event` TEXT,
`data` JSONB, `created_at` TIMESTAMPTZ.

## Module contract
(technical-spec.md → db.py) — all access uses an asyncpg pool; **no SQL anywhere
else in the project.**

Pool lifecycle:
- `init_pool() -> None` — min 2, max 10 connections. Raises on failure (caller
  exits non-zero).
- `close_pool() -> None` — called in the shutdown finally block.

Users:
- `get_user(user_id: int) -> dict | None` — full row or None. Fields:
  `telegram_user_id`, `todoist_token` (encrypted), `language`, `created_at`,
  `last_active_at`. No separate `get_user_token`.
- `save_user(user_id: int, encrypted_token: str, language: str) -> None` —
  insert or update; on conflict update token, language, `last_active_at`.
- `update_language(user_id: int, language: str) -> None` — updates `language`
  AND `last_active_at`.
- `touch_user(user_id: int) -> None` — updates only `last_active_at`.
- `get_all_users() -> list[dict]` — key-rotation script only.
- `update_token(user_id: int, encrypted_token: str) -> None` — key-rotation
  script only.

Conversations:
- `get_history(user_id: int) -> list[dict]` — `[row["content"] for row in rows]`,
  `created_at` ascending, excluding `role = "system"`. Raw JSONB objects.
- `save_user_message(user_id: int, content: dict) -> None` — inserts user row.
  Does NOT trim. Called at the start of every turn (including early returns).
- `save_turn(user_id: int, assistant_content: dict) -> None` — inserts assistant
  row and runs the `MAX_HISTORY_MESSAGES` trim **exactly once, atomically**. THE
  ONLY place trimming occurs. Trim counts all roles (user/assistant/tool).
- `save_tool_result(user_id: int, tool_content: dict) -> None` — inserts tool
  row. Does NOT trim.
- `delete_turn_tool_results(user_id: int, since: datetime) -> None` — deletes
  `role = "tool"` rows where `created_at >= since` (boundary inclusive by
  design). `since` MUST be timezone-aware — asyncpg rejects naive datetimes for
  TIMESTAMPTZ. Safe to call with no matching rows.
- `clear_user_history(user_id: int) -> int` — deletes all conversation rows for
  the user (all roles). Returns count (0 means none existed).
- `cleanup_old_conversations(days: int) -> int` — `WHERE created_at < NOW() -
  make_interval(days => $1)`. Integer bind param, no string interpolation.
  Returns rows deleted.
- `cleanup_old_logs(days: int) -> int` — same pattern for logs.

Logging:
- `log(user_id: int | None, level: str, event: str, data: dict) -> None` —
  sanitizes via `logger.sanitize`, inserts a logs row, calls `logger.log_stdout`.
  On insert failure: calls `log_stdout` only. Never propagates.

Content structure (`role` column always equals `content.role`):
- user: `{"role": "user", "content": "string"}`
- assistant (text): `{"role": "assistant", "content": "string"}`
- assistant (tool call): `{"role": "assistant", "content": null, "tool_calls": [...]}`
- tool: `{"role": "tool", "tool_call_id": "string", "content": "string"}`

## Relevant error handling rules
(technical-spec.md → Error Handling Rules — only those touching this module)

- **Rule 2** — DB log failures → `log_stdout` only. `log()` must swallow insert
  errors and still call `log_stdout`; it can never raise.
- **Rule 6** — On MCP failure or `save_tool_result` failure the caller calls
  `delete_turn_tool_results` and returns `tool_failure`; `delete_turn_tool_results`
  must therefore be safe to call mid-turn and with no matching rows.
- **Rule 12** — `init_pool` failure → caller logs to stdout and exits non-zero,
  so `init_pool` must raise on failure rather than degrade silently.
- **Rule 14** — Early returns rely on `save_user_message` having persisted user
  context and tool results being cleaned up; honor the "save_user_message never
  trims" / "save_turn trims once" split exactly.
- **Rule 15** — `db.get_user` exceptions are handled by the caller (logs ERROR,
  sends `db_error`); `get_user` should surface real failures, not mask them.

## Test cases
(technical-spec.md → tests/test_db.py) — real Postgres via `TEST_DATABASE_URL`
(never production); per-test transaction rolled back in teardown; migrations run
once via a session-scoped fixture.

- `get_user` / `save_user`: `get_user(999999)` → None; save then get returns the
  row; save twice with same id updates token and language; `last_active_at`
  updated on conflict.
- `update_language`: updates `language` and `last_active_at`; does not change
  `todoist_token`.
- `touch_user`: updates only `last_active_at`; leaves `language`/`todoist_token`.
- `save_user_message` / `save_turn` / trim: after `save_user_message` one user
  row exists and no trim; after `save_turn` assistant row exists and trim ran
  exactly once; insert 25 user+assistant pairs → exactly `MAX_HISTORY_MESSAGES`
  rows remain; trim runs in `save_turn` only; `save_turn` preserves most recent
  rows.
- `save_tool_result`: inserts `role = "tool"` row; does not trim.
- `delete_turn_tool_results`: deletes only `role = "tool"` rows with
  `created_at >= since`; does not delete user/assistant rows; does not delete
  tool rows created before `since`; inclusive-boundary row (created at exactly
  `since`) IS deleted; passing a naive datetime raises; no tool rows → silently
  deletes zero, no error.
- `get_history`: rows in `created_at` ascending order; excludes `role = "system"`;
  returns raw JSONB content objects (list of dicts with `role`); `[]` for new
  user.
- `clear_user_history`: returns 0 for user with no history; correct count after
  inserting; deletes all roles.
- `cleanup_old_conversations` / `cleanup_old_logs`: deletes rows older than
  `days`; preserves newer rows; returns correct count.
- `log`: inserts sanitized row into logs; calls `log_stdout` (mock, assert
  called); does not raise on DB failure (mock db to raise, assert no exception).
- `get_all_users` / `update_token` (key-rotation functions — the spec lists no
  cases for these, but they must be covered to meet the 80% threshold): write
  basic round-trips — `get_all_users()` returns all saved user rows (and `[]`
  when none exist); `update_token(user_id, new_encrypted)` changes only
  `todoist_token`, leaving `language` and `last_active_at` untouched.
